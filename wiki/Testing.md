# Testing

LocalDrop has tests at three levels: module unit tests, app unit tests, and UI tests. Tests are run with `xcodebuild` or `swift test` depending on the target.

## Test Targets

| Target | Path | Framework | Purpose |
|--------|------|-----------|---------|
| LocalSendKitTests | `Modules/LocalSendKit/Tests/LocalSendKitTests` | XCTest | Unit tests for LocalSendKit networking, crypto, models, server, client, discovery, and integration. |
| FeatureTransferTests | `Modules/FeatureTransfer/Tests/FeatureTransferTests` | XCTest | Unit tests for FeatureTransfer UI state and business logic. |
| AppLoggingTests | `Modules/AppLogging/Tests/AppLoggingTests` | XCTest | Unit tests for logging, redaction, sinks, and rotation. |
| LocalDropAppTests | `Tests/LocalDropAppTests` | XCTest | Tests for the app target helper `SendEntryPresentationState`. |
| LocalDropAppUITests | `Tests/LocalDropAppUITests` | XCTest | UI tests for menu commands, keyboard shortcuts, and seeded staged batches. |

## Running Tests

### Module Tests

Each module is a Swift Package, so you can run module tests with `swift test` from the module directory:

```bash
cd Modules/LocalSendKit && swift test
cd Modules/FeatureTransfer && swift test
cd Modules/AppLogging && swift test
```

### App Tests

The app target is generated by XcodeGen from `project.yml`. Open or generate the project, then run tests with `xcodebuild`:

```bash
xcodegen generate
xcodebuild -project LocalDrop.xcodeproj -scheme LocalDrop -destination 'platform=macOS' test
```

## LocalSendKit Tests

- `DiscoveryAndSessionTests.swift`: discovery flow and session management.
- `ClientAndSessionCoverageTests.swift`: client API coverage and session edge cases.
- `DiscoveryRuntimeCoverageTests.swift`: runtime discovery coverage.
- `ServerTests.swift`: HTTP server routing and request handling.
- `HTTPTests.swift`: HTTP parser and writer behavior.
- `CryptoTests.swift`: certificate generation, fingerprint, and validation.
- `ModelTests.swift`: Codable DTO round-trips and defaults.
- `NetworkRuntimeCoverageTests.swift`: network runtime coverage paths.
- `IntegrationTests.swift`: end-to-end transfer scenarios using in-process transport.

LocalSendKit uses an `InProcessTransport` for most tests so the HTTP client and server can be exercised without real network sockets.

## FeatureTransfer Tests

- `FeatureTransferTests.swift`: tests for `TransferFeatureStore`, `TransferFeatureContainer`, and adapter behavior.

The module exposes a `testing()` container with a no-op runtime so view logic can be tested without a network.

## UI Tests

`LocalDropMenuCommandUITests.swift` launches the app with `--ui-testing` and verifies:

- Command+1/2/3/4 navigate between Receive, Send, History, and Settings.
- Command+, opens Settings.
- Command+T opens the Send Text sheet.
- Settings reveal and regenerate the incoming PIN.
- Seeded batch shows staged items and supports removing one.

The app supports the following launch arguments for UI testing:

- `--ui-testing`: enables the testing container.
- `--ui-testing-incoming-pin-enabled`: enables the incoming PIN.
- `--ui-testing-seed-staged-batch`: pre-stages three sample files.

## Localization Verification

The script `scripts/verify-featuretransfer-localizations.swift` checks that every key in `Localizable.xcstrings` has an English value and that no unused keys remain. Run it from the repository root.

```bash
swift scripts/verify-featuretransfer-localizations.swift
```

## Verification Script

`scripts/verify-localdrop.sh` runs the project-level smoke checks. Inspect the script for the exact commands it runs.
